# Meliora Roadmap

本路线图用于记录接下来计划推进的功能方向。版本内容会随实际开发、测试结果和用户反馈调整;已完成内容以 `CHANGELOG.md` 为准。

## 0.2.1

### S3 存储支持

- 在管理后台新增对象存储配置入口,支持 S3 兼容服务的 Endpoint、Bucket、Region、Access Key、Secret Key 与公开访问域名配置。
- 本地音乐上传支持选择 GitHub 仓库写入或 S3 写入,并在曲库配置中保存最终可公开访问的音频、封面、歌词 URL。
- 文件删除、覆盖、重命名等管理操作同步支持 S3 对象清理,避免后台配置与对象存储内容不一致。
- S3 密钥仅保存在后端加密配置中,不得进入前端 bundle 或公开配置。
- 覆盖 Cloudflare R2、AWS S3、MinIO 等常见 S3 兼容服务的配置差异与错误提示。

### 酷狗音乐源支持

- 在远程歌单配置中新增酷狗音乐源选项,与现有网易云、QQ 音乐入口保持一致。
- 扩展音乐源类型、配置校验、后台编辑器与 Meting/API 请求参数映射。
- 增加酷狗歌单返回数据的适配与归一化测试,确保标题、歌手、封面、音频、歌词字段稳定落到统一 `Track` 结构。
- 对酷狗源不可用、接口限流、缺少歌词或音频地址等情况提供单源失败降级,不影响其他音乐源和本地曲目加载。

## 后续候选方向

### 播放器架构整理

- 继续从 `src/views/PlayerView.vue` 中拆出复杂业务逻辑,让播放器页面回到编排层。
- 评估 `useAudioPlayer` 状态机重构,降低切歌、预加载、错误恢复之间的隐式状态耦合。

### 性能与资源占用

- 继续验证预加载、封面解码、主题色提取、歌词滚动在大曲库与移动端下的表现。
- 完成 `.trae/specs/reduce-memory-gpu-usage` 中剩余验收项。
- 针对 200+ 曲目场景持续完善虚拟滚动与搜索性能。

### 部署与本地开发体验

- 完善本地生产近似测试流程,让不配置真实 GitHub 仓库时也能通过重新生成公开配置验证主页表现。
- 补充 S3/R2 部署文档、环境变量说明和常见错误排查。
- 保持 Vercel、Netlify、Cloudflare Pages 三个平台配置一致。
